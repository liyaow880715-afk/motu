# 摹图 — AI 设置与生成详情页教程

> 本教程指导你从零开始，完成 AI Provider 配置、商品分析、页面规划、图片生成到导出的完整流程。

---

## 📋 前置准备

1. **启动项目**：`npm run dev`，打开浏览器访问 `http://localhost:3000`
2. **获取激活码**：联系管理员获取激活码（支持次卡 / 日卡 / 月卡）
3. **准备商品素材**：至少 1 张商品主图（建议 3-5 张不同角度）

---

## 第一步：配置 AI Provider（关键！）

摹图需要接入一个 **OpenAI-compatible** 的 API 才能工作。支持 Gemini、OpenAI、Claude、以及任意兼容代理。

### 1.1 进入设置页面

点击页面右上角的 **⚙️ 设置** 图标 → 选择 **模型接入** 标签。

### 1.2 填写 Provider 信息

| 字段 | 说明 | 示例 |
|------|------|------|
| **名称** | 给这个配置起个名字 | "Gemini 代理" |
| **Base URL** | API 的基础地址 | `https://api.yijiarj.cn/v1` |
| **API Key** | 你的 API 密钥 | `sk-xxxxxxxx` |

> 💡 **没有 API Key？**
> - 国内用户可使用 Gemini 代理（如 `https://api.yijiarj.cn/v1`）
> - 有 OpenAI 账号可直接使用 `https://api.openai.com/v1`
> - 也可使用其他兼容代理，如火山引擎、硅基流动等

### 1.3 发现模型

填写完成后点击 **发现模型** 按钮，系统会自动探测该 Provider 支持的模型列表。

### 1.4 设置默认模型

探测完成后，为不同任务分配默认模型：

| 任务类型 | 推荐模型 | 说明 |
|----------|----------|------|
| **分析模型** | 带 vision 能力的模型 | 如 `gemini-2.5-pro-exp-03-25`、`gpt-4o` |
| **规划模型** | 文本能力强的模型 | 同上，或 `claude-sonnet-4-20250514` |
| **头图模型** | 支持图片生成的模型 | 如 `gemini-2.0-flash-exp-image-generation` |
| **详情图模型** | 支持图片生成的模型 | 同上 |
| **图片编辑模型** | 支持图片编辑的模型 | 同上 |

> ⚠️ **注意**：图片生成和编辑必须使用支持图像接口的模型。纯文本模型（如 GPT-4）只能做分析和规划，不能生图。

### 1.5 保存配置

点击 **保存配置**。保存后该 Provider 会被激活，后续所有 AI 任务都会使用它。

---

## 第二步：创建项目

### 2.1 输入商品信息

在首页（快速开始）填写：

- **产品信息**：商品的核心描述（如"夏季新款法式碎花连衣裙，V领收腰设计"）
- **商品分类**（可选）：如"女装 / 连衣裙"
- **核心卖点**（可选）：如"透气雪纺面料、显瘦收腰、可机洗"
- **目标人群**（可选）：如"25-35岁都市白领"

### 2.2 上传商品图片

拖拽或点击上传至少 **1 张主图**。建议上传：
- 1 张商品正面全貌图
- 1-2 张细节图（面料、工艺特写）
- 1 张场景/搭配图（如果有）

> 📌 **图片会被自动压缩**：超过 1.5MB 的图片不会被发送到 AI 分析，系统会自动选择合适大小的图片。

### 2.3 点击"开始生成详情页方案"

系统会创建项目并跳转到 **分析页面**。

---

## 第三步：AI 商品分析

### 3.1 启动分析

在分析页面点击 **开始 AI 分析**。系统会调用分析模型，基于你提供的图片和文字进行深度理解。

分析过程通常需要 **30-120 秒**，取决于模型响应速度。

### 3.2 查看分析结果

分析完成后，页面会显示结构化结果：

| 字段 | 说明 |
|------|------|
| **产品名称** | AI 提炼的商品名称 |
| **分类 / 子分类** | 商品所属品类 |
| **风格标签** | 如"简约、优雅、通勤" |
| **目标人群** | 消费群体画像 |
| **核心卖点** | AI 提取的 4-6 个关键卖点 |
| **差异化亮点** | 与竞品的区别 |
| **使用场景** | 适用场合 |

### 3.3 调整输出配置

在分析页面右下角的 **输出配置** 卡片中，可以设置：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **头图数量** | 4 张 | 详情页顶部的头图张数（1:1 比例） |
| **详情页数量** | 6 张 | 详情模块数量 |
| **详情图比例** | 9:16 | 详情图的宽高比（可选 3:4 或 9:16） |
| **内容语言** | 中文 | 文案语言 |

> 🔔 **服装类产品特殊说明**：
> - 如果 AI 检测到服装类（如女装、男装、T恤、裙子等），系统会自动使用 **11 个固定详情模块模板**
> - 详情图比例会被强制设为 **3:4**
> - 规划页面会显示提示："检测到服装类产品，将自动使用固定详情页模板"

### 3.4 保存分析结果

确认分析结果和输出配置无误后，点击 **保存分析结果**。

---

## 第四步：页面规划

### 4.1 进入规划页面

点击左侧导航的 **规划** 标签，或从分析页面点击 **下一步：页面规划**。

### 4.2 检查输出配置

规划页面第 1 步会显示从分析页继承的配置：
- 内容语言
- 头图目标（1:1 比例）
- 详情页目标数量
- 详情图比例

如果需调整，点击 **返回分析页调整配置**。

### 4.3 选择模特（服装类可选）

如果是服装类产品，规划页面会显示 **全局模特选择** 区域：

1. 点击下拉框，从模特库中选择一位模特
2. 系统会在生成包含人物的模块时，自动参考该模特保持人物一致性
3. 选择后点击 **保存生成设置**

> 📌 模特库需要在 **模特库** 页面提前上传模特的三视图（正面、背面、侧面）。

### 4.4 启动 AI 规划

点击 **开始 AI 规划**。系统会根据商品分析结果，为每个模块生成：

- **模块标题**：如"法式碎花连衣裙·模特全身展示"
- **模块目标**：该模块的设计目的
- **营销文案**：包含主标题和卖点 bullet
- **双语视觉提示词**：中文提示 + English Prompt，用于 AI 生图

规划完成后，页面会显示完整的模块列表：
- **头图模块**（HERO）：顶部展示图
- **详情模块**：根据商品类型自动分配不同类型

### 4.5 服装类固定模块（自动）

服装类产品规划后会得到以下 11 个固定详情模块：

| 序号 | 模块名称 | 类型 | 说明 |
|------|----------|------|------|
| 1 | 模特全身展示 | 场景图 | 模特站立全身照 |
| 2 | 产品正反面展示 | 卖点图 | 服装正反面平铺/挂拍 |
| 3 | 设计细节展示 | 特写图 | 领口、袖口、纽扣等工艺 |
| 4 | 材质展示 | 材质图 | 面料纹理、光泽特写 |
| 5 | 尺码表 | 规格图 | 尺码对照表 |
| 6 | 模特正面展示 | 场景图 | 模特正面照 |
| 7 | 模特背面展示 | 场景图 | 模特背面照 |
| 8 | 模特侧身展示 | 场景图 | 模特侧身照 |
| 9 | 产品正面大图 | 特写图 | 正面高清全貌 |
| 10 | 产品背面大图 | 特写图 | 背面高清全貌 |
| 11 | 产品细节大图 | 特写图 | 关键细节大图 |

---

## 第五步：编辑与生成图片

### 5.1 进入编辑器

规划完成后，点击左侧导航的 **编辑** 标签，进入编辑器。

编辑器三栏布局：
- **左栏**：模块结构树（头图 + 详情模块）
- **中栏**：手机预览（实时查看效果）
- **右栏**：模块编辑面板

### 5.2 编辑模块内容

在右栏编辑面板中，可以修改：

| 字段 | 说明 |
|------|------|
| **类型** | 模块类型（场景图、卖点图、特写图等） |
| **标题** | 模块名称 |
| **模块目标** | 设计目的说明 |
| **模块文案** | 营销文案 |
| **双语视觉 Prompt** | AI 生图提示词 |

> 💡 **Prompt 格式要求**：
> - 必须包含 `中文提示：... English Prompt: ...`
> - 头图模块使用 `1:1` 比例描述
> - 详情模块使用 `3:4` 或 `9:16` 比例描述
> - 服装类模特模块需包含模特一致性语句

### 5.3 上传参考图（可选）

为单个模块上传专属参考图：
1. 在右栏点击 **上传参考图**
2. 选择图片，该图会专用于当前模块的 AI 生成
3. 也可在 **全局参考图** 区域勾选已上传的商品图作为额外参考

### 5.4 生成图片

编辑完成后，点击 **生成当前模块图**（首次）或 **重新生成当前图**（已有图）。

系统会：
1. 将模块的 visualPrompt、文案、参考图发送给 AI
2. AI 生成对应图片
3. 自动保存到当前项目和版本历史

> ⏱️ **生成时间**：每张图约 30-90 秒，取决于模型和网络。

### 5.5 版本管理

每个模块支持多版本：
- 每次重新生成会创建一个新版本
- 在 **版本历史** 区域可以查看所有版本
- 点击 **设为当前** 切换生效版本

### 5.6 批量生成

规划页面支持 **一键批量出图**：
1. 回到规划页面
2. 点击 **一键批量出图**
3. 系统会按顺序为所有未生成图片的模块调用 AI

---

## 第六步：导出详情页

### 6.1 进入导出页面

所有模块生成完成后，点击左侧导航的 **导出** 标签。

### 6.2 选择导出格式

目前支持：
- **长图导出**：将所有模块拼接为一张长图
- **DOCX 导出**：导出为 Word 文档（含图片和文案）

### 6.3 设置导出参数

| 参数 | 说明 |
|------|------|
| **页面宽度** | 默认 750px（移动端标准） |
| **图片质量** | 高清 / 标准 |
| **背景色** | 白色 / 自定义 |

### 6.4 执行导出

点击 **导出**，系统会生成文件并提供下载。

## 🎽 服装类全流程示例

### 示例：生成一件连衣裙的详情页

**Step 1 — 创建项目**
```
产品信息：夏季新款法式碎花连衣裙，V领收腰设计，雪纺面料
分类：女装 / 连衣裙
卖点：透气凉爽、显瘦收腰、法式优雅、可机洗
```

**Step 2 — 上传图片**
- 上传连衣裙正面图、背面图、面料特写图

**Step 3 — AI 分析**
- 系统自动识别为"服装类"
- 提取风格标签：法式、碎花、优雅、通勤

**Step 4 — 规划（自动）**
- 系统提示："检测到服装类产品"
- 自动生成 11 个固定模块 + 4 张头图
- 详情图比例强制 3:4

**Step 5 — 选择模特**
- 从模特库选择一位亚洲女性模特
- 系统会在模特展示模块中注入该模特的参考图

**Step 6 — 生成图片**
- 模特全身/正面/背面/侧身模块：AI 参考模特三视图生成
- 产品正反面/细节模块：AI 参考商品图生成
- 尺码表模块：AI 生成排版图

**Step 7 — 导出**
- 导出为长图，用于淘宝/抖音/小红书等电商平台

## ❓ 常见问题

### Q1: 分析一直超时怎么办？

- 检查 Provider 配置是否正确（Base URL 和 API Key）
- 尝试切换到响应更快的模型（如 GPT-4o 替代 Gemini）
- 检查 API Key 是否有额度

### Q2: 图片生成失败怎么办？

- 确认选择的模型支持图片生成（需带 `image_gen` 或 `image_edit` 能力）
- 检查模型探测结果：在 Provider 设置中查看该模型的 **图片生成端点** 状态
- 如果显示 "unavailable"，说明该模型在此 Provider 上不支持生图，需换模型

### Q3: 生成的图片不像我的商品？

- 确保上传了清晰的商品主图
- 在编辑器中为模块勾选 **全局参考图**
- 上传 **专属参考图** 到具体模块
- 修改 visualPrompt，加入更具体的商品描述

### Q4: 服装类没有自动检测？

- 检查分析结果中的 **分类** 和 **子分类** 是否包含服装关键词
- 支持的关键词：服装、女装、男装、童装、上衣、T恤、衬衫、裤子、裙、外套等
- 如未识别，可手动在分析页面修改分类后重新保存

### Q5: 如何更换 AI Provider？

- 进入 **设置 → 模型接入**
- 可以保存多个 Provider 配置
- 点击列表中的 **激活** 按钮切换当前使用的 Provider
